# AI News And Social Sharing Automation

This project keeps manual blog publishing and automated AI-news publishing as
separate streams.

- Manual blogs are still written and managed through WordPress.
- Automated AI news is generated by the daily cron job and saved locally in
  Vercel Blob.
- When both streams are shown together, manual WordPress blogs stay before the
  automated news entries.

## What Runs Daily

Vercel Cron calls:

```text
GET /api/cron/ai-news
```

Schedule:

```text
0 1 * * *
```

That is 01:00 UTC, or 09:00 in Taipei.

The job:

1. Reads AI/startup RSS feeds.
2. Uses Gemini by default to parse the sources and generate a cited automated
   news article in English, Traditional Chinese, and Vietnamese.
3. Stores the generated news entries in Vercel Blob at `blog/posts.json`.
4. Keeps WordPress/manual blogs ahead of generated news in combined blog lists.
5. Optionally shares the new automated news entries through Postiz and/or LINE.

## Required Vercel Environment Variables

Gemini is the default provider:

```env
BLOB_READ_WRITE_TOKEN=...
CRON_SECRET=...
GEMINI_API_KEY=...
AI_NEWS_PROVIDER=gemini
AI_NEWS_MODEL=gemini-3.5-flash
AI_NEWS_LANGUAGES=en,zh,vi
```

`GOOGLE_GENERATIVE_AI_API_KEY` is also accepted if that is the existing Gemini
key name in the deployment.

To use OpenAI instead:

```env
AI_NEWS_PROVIDER=openai
OPENAI_API_KEY=...
AI_NEWS_MODEL=gpt-5.5
```

To use Claude instead:

```env
AI_NEWS_PROVIDER=anthropic
ANTHROPIC_API_KEY=...
AI_NEWS_MODEL=claude-sonnet-4-5
```

To show only generated automated news and stop reading WordPress blog posts:

```env
BLOG_SOURCE=local
```

Leave `BLOG_SOURCE` unset when you still want your manually written WordPress
blogs to appear.

To customize news sources:

```env
AI_NEWS_FEEDS=https://example.com/feed.xml,https://another.com/rss
```

To generate only one language during a manual run, pass `lang=en`, `lang=zh`,
or `lang=vi`. To generate a custom set, pass `languages=en,zh,vi`.

## Manual Blogs Vs Automated News

Manual blogs remain the primary blog content. The automated news job does not
replace the manual blog workflow unless `BLOG_SOURCE=local` is set.

Generated news entries use:

- `source: ai-news-agent`
- `category: Automated News`
- `language: en`, `language: zh`, or `language: vi`
- Slugs starting with `ai-industry-brief-YYYY-MM-DD-language-`

This keeps generated news identifiable even though it is rendered through the
same `/blog/[slug]` page as manual posts.

## Why Postiz + LINE

Use Postiz for Facebook, Instagram, X, Threads, LinkedIn, Bluesky, Mastodon,
and similar Western/Taiwan business channels. Postiz is open-source,
self-hostable, and uses official OAuth flows instead of browser scraping.

Use LINE Messaging API separately because LINE sharing is done through a LINE
Official Account, not through normal social scheduler posting.

References:

- Postiz: https://github.com/gitroomhq/postiz-app
- Postiz Public API: https://docs.postiz.com/public-api/introduction
- LINE Messaging API: https://developers.line.biz/en/docs/messaging-api/sending-messages/

## Enable Social Sharing

Set:

```env
SOCIAL_SHARE_ENABLED=true
SOCIAL_SHARE_TARGETS=postiz,line
```

You can disable either target:

```env
SOCIAL_SHARE_TARGETS=postiz
SOCIAL_SHARE_TARGETS=line
```

Social sharing runs after each automated news entry is saved. If sharing fails,
the generated news entries remain published and the cron response includes
`socialShareResults` for debugging.

## Postiz Setup

1. Use Postiz Cloud or self-host Postiz.
2. Connect channels in Postiz with OAuth, such as Facebook Page, Instagram, X,
   Threads, and LinkedIn.
3. In Postiz, create a Public API key.
4. Get each connected channel's integration ID from the Postiz integrations API
   or dashboard.

Set Vercel env vars:

```env
POSTIZ_API_URL=https://api.postiz.com/public/v1
POSTIZ_API_KEY=your-postiz-api-key
POSTIZ_INTEGRATIONS=facebook-integration-id:facebook,instagram-integration-id:instagram,x-integration-id:x,threads-integration-id:threads,linkedin-id:linkedin
POSTIZ_POST_TYPE=now
POSTIZ_DEFAULT_MEDIA_URL=https://www.tecxmate.com/graphics/Strategy%20%26%20Research.png
```

`POSTIZ_INTEGRATIONS` also accepts JSON:

```json
[
  { "id": "facebook-integration-id", "type": "facebook" },
  { "id": "instagram-integration-id", "type": "instagram" },
  { "id": "x-integration-id", "type": "x" }
]
```

Use `POSTIZ_DEFAULT_MEDIA_URL` for Instagram because Instagram normally needs
media. Keep this URL public and directly fetchable.

For self-hosted Postiz, set:

```env
POSTIZ_API_URL=https://your-postiz-domain.example.com/public/v1
```

## LINE Setup

1. Create or use a LINE Official Account.
2. Enable Messaging API in LINE Developers.
3. Create a channel access token.
4. Add the account as a friend for tests.

Broadcast to all friends:

```env
LINE_CHANNEL_ACCESS_TOKEN=your-line-channel-access-token
```

Or push only to one user during tests:

```env
LINE_CHANNEL_ACCESS_TOKEN=your-line-channel-access-token
LINE_TARGET_USER_ID=line-user-id-for-test
```

## Manual Tests

Generate automated news in all configured languages without saving or sharing:

```bash
curl "https://www.tecxmate.com/api/cron/ai-news?dryRun=1&secret=$CRON_SECRET"
```

Generate, save, and share today's automated news in all configured languages:

```bash
curl "https://www.tecxmate.com/api/cron/ai-news?secret=$CRON_SECRET"
```

Generate only Traditional Chinese:

```bash
curl "https://www.tecxmate.com/api/cron/ai-news?lang=zh&secret=$CRON_SECRET"
```

Regenerate today's automated news, save each language to the same slug, and
share again:

```bash
curl "https://www.tecxmate.com/api/cron/ai-news?force=1&secret=$CRON_SECRET"
```

## Stability Notes

- Gemini is the default parser/generator for automated news.
- OpenAI and Claude remain available provider options through
  `AI_NEWS_PROVIDER`.
- Postiz is the best free/self-hostable match for Facebook, Instagram, X,
  Threads, and LinkedIn style posting.
- LINE is stable through its official Messaging API, but you need a LINE
  Official Account and must respect message quotas.
- X API access and Meta publishing permissions can require approval or paid API
  access depending on account/app status. Postiz reduces integration work but
  cannot bypass platform requirements.
